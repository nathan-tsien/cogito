# v0.1 JSONL Append Baseline (informational)

> Measured against `cogito-store-jsonl` via
> `crates/cogito-store-jsonl/benches/append_throughput.rs`.
>
> **This is a dev-grade backend baseline, not a production SLO.** The
> ADR-0005 §3 P99 < 5 ms target is locked at v0.4 against
> `cogito-store-postgres` under production-realistic load.

## Environment

- Host: `Linux cn0014004480l 5.15.0-139-generic #149~20.04.1-Ubuntu SMP Wed Apr 16 08:29:56 UTC 2025 x86_64 x86_64 x86_64 GNU/Linux`
- Date: 2026-05-19
- Rust: `rustc 1.92.0 (ded5c06cf 2025-12-08)`
- JSONL durability: userspace flush only (`tokio::fs::File::flush`); no `sync_data` / `fsync`.

## Results — `jsonl_append_single_event`

Criterion ran 100 samples across an estimated 429,000 iterations
(5.0 s collection window after 3.0 s warmup). Per-sample latencies
were derived from `target/criterion/jsonl_append_single_event/new/sample.json`
(`time_ns / iters` per sample).

| Metric | Value |
|---|---|
| Mean per-append latency | 11.41 µs (slope estimate 11.32 µs) |
| P50 latency | 11.63 µs |
| P99 latency | 13.43 µs |
| P99.9 latency | 16.15 µs (single-sample approximation; 100 samples) |
| Throughput (at mean) | ~87,600 events/sec |
| Sample event payload size | ~200 bytes (`ToolUseRecorded`) |

Criterion's own confidence-interval summary for the mean was
`[11.07 µs, 11.56 µs]` at 95 % confidence.

## Reproducing

```bash
just bench-baseline
```

Criterion HTML reports under `target/criterion/jsonl_append_single_event/`.
