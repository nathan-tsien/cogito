Tool 块 —— 完整生命周期的视觉与交互

Tool 从 dispatch → running → done (ok / err) → selected → expanded 五个状态。三种风格，每个都展示完整流程。

∴  Let me read that file...

   ⠋ read_file            0.3s
   ✓ grep                 45ms
   ✗ run_tests            2.1s
     ↳ panicked: assertion failed


   ▸ read_file            12ms
     ↑↓ navigate · enter expand


   ▾ read_file            12ms
     args   {"path": "main.rs"}
     ↳ pub fn main() {
     ↳     println!("hello");
     ↳ }

T1. Status Glyph

每个状态对应一个固定 glyph：⠋ running（动）、✓ ok、✗ err、▸ collapsed-selected、▾ expanded。错误信息直接缩进显示（红 ↳）。最 readable，glyph 一眼就知道状态。

∴  Let me read that file...

   ▎read_file             running 0.3s
   ▎grep                  ok 45ms
   ▎run_tests             err 2.1s
   ▎  panicked: assertion failed


   ▎read_file             ok 12ms
   ▎  ↑↓ navigate · enter expand


   ▎read_file             ok 12ms
   ▎  args   {"path": "main.rs"}
   ▎  ↳ pub fn main() {
   ▎  ↳     println!("hello");

T2. Color Spine

每个 tool 块左侧一条细色条 (▎)，颜色随状态变化：黄=running、灰=ok 完成、红=err、蓝=selected、绿=expanded。色条把多行内容自然连成一段，多个 tool 像 timeline 一样列开。

∴  Let me read that file...

   read_file  ⌜running 0.3s···⌝
   grep       ⌜done 45ms⌝
   run_tests  ⌜failed 2.1s⌝
     panicked: assertion failed


   read_file  ⌜done 12ms⌝  ◀ selected


   read_file  ⌜done 12ms⌝  ▼
     {"path": "main.rs"}
     pub fn main() {
         println!("hello");
     }

T3. Bracket Badge

状态信息用一个色块 ⌜...⌝ 包裹，紧跟 tool 名后面。无 glyph、无色条。状态色直接打在 badge 里，类似 GitHub Actions 的 step badge。最简洁的横排版式。

键盘交互（三套共用，但你可以提偏好）

默认建议

  • Ctrl-↑/↓ 在 tool 块间移动 selection
  • Enter 展开/折叠当前 selection
  • Esc 取消 selection 回到 input
  • 1-9 快速展开最近 N 个 tool（数字快捷）
  • e / c 展开/折叠当前消息的所有 tool

问题

  • Running tool 也可以被 select 吗？还是只能 select 已完成的？
  • Selection 焦点失去后（如 PgDn 滚动）保留还是清空？
  • 是否要 Mouse 支持（hover + click 展开）？v0.1 跳过？
