四组符号 —— 让 cogito 这个名字有点呼应

"cogito ergo sum" = "我思故我在"。Cogito 这个项目刻意把"思考"（Brain）和"行动"（Hands）分离 —— runtime 从不直接行动，它做决策。下面四组符号都顺着这个理念。

▸  who are you?

∴  I am cogito, an embedded
   agent runtime.

   ▸ read_file  ok
   ▾ grep  ok

▸  ship it then

∴  Done.

S1. ▸ / ∴  "therefore"

最直白的引用 —— ∴ 是数学/哲学的"therefore"，正是 "cogito ergo sum" 里那个推导关键字。每条 agent 回复都是一次"故…"的结论。文化梗 + 视觉极简 + 渲染兼容性好。

›  who are you?

◇  I am cogito, an embedded
   agent runtime.

   ▸ read_file  ok
   ▾ grep  ok

›  ship it then

◇  Done.

S2. › / ◇  "open mind"

抽象 / 开放 —— ◇ 空心菱形 = 待填充的思维空间、未决断的开放结构。› 是轻量 prompt 标记。比 S1 更克制、更通用，不依赖文化梗。

▸  who are you?

※  I am cogito, an embedded
   agent runtime.

   ▸ read_file  ok
   ▾ grep  ok

▸  ship it then

※  Done.

S3. ▸ / ※  "annotation"

引注 / 标注 —— ※ 是引用 / 旁注的符号，呼应 "cogito 是给系统加注解的大脑" —— Hands 做事，Brain 在旁边标注 / 决策。带书卷气。

▸  who are you?

◈  I am cogito, an embedded
   agent runtime.

   ▸ read_file  ok
   ▾ grep  ok

▸  ship it then

◈  Done.

S4. ▸ / ◈  "nested mind"

嵌套结构 —— ◈ 菱形中含菱形，对应 cogito 的分层架构（Brain → Runtime → Hands）。"思考之内还有思考" 的层级感。视觉上比 S1/S2 更厚重。
